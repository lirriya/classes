Basics of Memory

What is a bit and a byte?

  • 1 bit = a binary digit, so it can be either 0 or 1
  • 1 byte (aka 1 octet in French) = 8 bits. So something like 00111010
  • 1 byte = 1 unit of memory

What is RAM?

The main memory (aka RAM or mémoire vive in french) of a computer is made of billions of memory cells, where each cell stores 1 but and there cells are grouped into bytes(8 bits) for practical use

Memory is addressed

Each byte (octet) has a unique address, we can think of it as a house number, which is basically just a number

If a memory has 256 bytes, the addresses can go from 0 to 255

So what happens when we declare a variable in C?

First let's remember what Declaration is. Declaration of a variable is when we announce that now we have this variable with this particular type (usually int, string(char[] in C), float, char). It looks sommething like this int a;

So what happens behind the scenes?

  1. The size is determined (we'll talk about it more indepth later). Int usually takes 4 bytes(32bits)
  2. A memory zone is reserved, for example: Adresses 0xFF00, 0xFF01, 0xFF02, 0xFF03 are all reserved for out int a
  3. A link is made between the variable name and the memory address in a register(which is a temporary storage in the CPU). So we can say that a refers to memory starting at 0xFF00

How can we access or use the said varirable? well quite easy. We can either use the variable directly by it's name, in our case "a".

Or we can use its address with a pointer (more about pointers later!)

Variables

So what is the variable? A variable is a memory slot created by the system to store a value.

Each variable has its own:

  • Type (int, char, float, etc)
  • Name (a, b, c, etc)
  • Value (1, 2, 3, etc)
  • Address (0xFF00, 0xFF01, etc)

>Basic Data Types

  • char: A single character: 1 byte (octet)
  • int: An integer: 4 bytes (octet)
  • float: A decimal number: 4 bytes (octet)
  • double: A more precise float: 8 bytes

Characters char

A char stores the ASCII code of a character

so for "A" = 65, and for "a" = 97, etc

So basically it holds an integer (0 to 255)

char c = 'A'; // stores 65 in memory

Integers

signed char: size 1 byte

unsigned char: size 1 byte

short int: size 2 bytes

unsigned short int: size 2 bytes

int: size 4 bytes

unsigned int: size 4 bytes

unsigned long int: 8 bytes

Floating-point numbers

float: size 4 bytes

double: size 8 bytes

Exercise 1:

Context: We have 32 Mo(megabytes) of memory.

We should remember that 1 Mo = 1 megaoctet = 1 million octets(bytes), but in computer memory, we use powers of 2.

So 1 Mo = 2^20 octets = 1.048.576 octets

So 32 Mo = 32 * 2^20 = 33.554.432 octets

Q1:How many char can it store?

So lets walk through and approach it with logical thinking. "How big is a char?"

We know that unsigned and signed char both are the size of 1 byte. So if 32 Mo is equal to 33,554,432 bytes, then we can store exactly 33, 554,432 chars.

Q2:How many int can it store?

So with the same logic. Size of int is 4 bytes. We can reconstruct the question to "How many 4-byte chunks can i fit in 33,554,432 bytes?". We just do 33,554,432/4=8388608

Q3:How many float can it store?

The size of float is the same as an int, so the answer is the same.

Q4: How many bits are needed to address all of memory.

This is a power of 2 problem.

We put 2 in a power of n >= 33,554,432. We do calculations and we get 2 in a power of 24. So the answer is: we need 25 bits to address this memory

Exercise 2:

Context: a variable int a is stored at address 0xFF00

Q1What are the addresses used by a?

Think: "How many bytes does an int take?". The answer is 4 bytes. So "if a starts at 0xFF00, what are the next 4 consecutive addresses?" So we start at 0xFF00 and move forward.

The answer: 0xFF00, 0XFF01, 0XFF02 and 0XFF03

Q2What is the address right after a?

We already used from 0xFF00 to 0xFF03, so the next free address is 0xFF04.

Q3What is the maximum memory size of this computer?

Here the answer lies in address. We were given that a is stored at address 0xFF00.

Think: "What's the highest possible addressable memory if addresses go up to 0xFFFF?". So basicaly 0xFFFF in decimal is 65,535 bytes. So by using the exercise before, we can find that it's 2 in a power of 16 so it means 64Ko.
